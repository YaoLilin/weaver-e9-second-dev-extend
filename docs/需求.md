
# 优化
- 需在 WorkflowActionParamParser 类中说明获取参数的规则，参数对象中什么属性才被作为参数。

# 实现 Action 参数对象生成
- 完善WorkflowApiParamValueInjector类，实现根据传入的参数类型，创建该参数对象，并根据配置的Action参数中的赋值，对该参数对象中的属性以及嵌套对象进行赋值。
## 赋值规则
- 当参数对象中的属性为基本类型时，包括 List<?>（? 代表基本类型，如 String、Integer ）, 直接对该属性进行赋值
- 如果参数属性中属于内嵌的对象，并且使用 @ActionParam 注解标记，包括 List<?> （? 代表非基本类型，必需使用 @ActionParam 注解标记），
需对内嵌的对象中内的属性字段进行赋值
- 赋值配置可从 uf_action_param 表中获取，在 WorkflowActionParamExtendAction.saveConfig 中已保存赋值配置
### 赋值方式为表单字段
- 对于非数组类型的参数，如果选择的是主表字段，则使用 WorkflowFieldValueFetchInterface 获取主表字段的值，如果选的是明细表字段，则获取
明细表中的字段的值，如果明细表有多行，则将多行的值合并，使用逗号分隔，前提是参数的类型必需为字符串类型，否则抛出异常。
- 对于数组类型的参数（List），并且数组中的元素类型为基本类型，如果选择的是主表字段，则数组只有1个元素，元素为主表字段的值，如果选择的是明细表
字段，则数组的元素为从明细表获取的字段的值，明细表数据有多行，则数组有多个元素。需对取到的字段的值进行转换，如果数组元素类型为 Integer, 则
需要将取到的字段值转换为 Integer ，如果无法转换，则抛出异常，获取到的字段值为空时不进行转换，不放入数组中。
- 对于数组类型的参数（List），并且数组中的元素类型为对象（非基本类型），则生成该数组的元素对象。
  - 如果元素对象内的属性的赋值选择为主表，则直接 取主表中的值
  - 如果选择的是明细表，则校验选择的明细表字段是否与当前数组类型的参数所配置的明细表一致，必需为同一个明细表，否则抛出异常
  - 根据元素对象内属性选择的明细表字段，获取明细表数据生成该元素对象，每条明细表数据生成一个对象，每个元素对象表示1条明细表数据
  - 数组参数配置的明细表可从 uf_action_param 表中的 detail_table 字段获取，该字段存储明细表序号，序号从1开始。
- 对于内嵌的对象中的属性，或者List中的元素对象中的属性，也适用于上述规则，比如List元素中的属性有 List 字段，适用于上述的List字段的处理。

### 赋值方式为系统参数
- 完善 WorkflowSystemParamProvider 类，获取系统参数
- 找到对应的SystemParamEnum枚举类型，只有属于这些枚举才能获取系统参数，否则抛出异常。
- 根据对应的枚举获取系统参数，可使用 WorkflowFieldValueFetchInterface 的 getSystemFieldValue 方法获取系统参数，对于 workflowId 
可使用 WorkflowUtil 进行获取
- 如果参数类型为数组，并且元素为基本类型，则获取到系统参数的值后生成一个List元素，放入数组中，如果元素类型为数字，则需要将系统参数的值转换为数字，
如果转换失败则抛出异常

### 赋值方式为固定值
- 直接取 uf_action_param 表中的赋值字段（assignment_value字段），赋值到属性中
- 如果属性为 List，并且元素类型为字符串，则获取固定值生成一个元素存入到List中，如果为数字类型，则将固定值转换为数字，如果转换失败则抛出转换异常
